# Nicolas Cole - Frameworks e Metodologias

## Framework 1: O 1-1-1 Framework

### O que é
A regra mais importante de Cole para qualquer peça de conteúdo: 1 problema do leitor, 1 solução específica, 1 ação concreta. Tudo que vai além disso é ruído.

### Como funciona
```
ANTES DE ESCREVER — A verificação:
├── Qual é o 1 problema que este texto resolve?
│   (Se você não sabe, o texto não está pronto)
├── Qual é a 1 solução que você está oferecendo?
│   (Não 5 soluções — 1, executável)
└── Qual é a 1 ação que o leitor deve tomar após ler?
    (Específica e imediata, não "reflita sobre isso")

ESTRUTURA RESULTANTE:
├── Abertura: O problema (2-3 frases)
├── Diagnóstico: Por que o problema existe (2-3 parágrafos)
├── Solução: O que fazer (o mais curto possível)
└── Ação: O próximo passo específico (1-2 frases)

TESTE FINAL:
└── Complete a frase: "Este texto ajuda [PERSONA] a [RESULTADO] fazendo [AÇÃO]."
    Se não consegue completar, volte ao início.
```

### Quando usar
- Qualquer peça de conteúdo digital (post, email, newsletter)
- Copywriting de landing page
- Scripts de vídeo curto

## Framework 2: Os Big 5 Tipos de Conteúdo

### O que é
Cole identificou 5 categorias de conteúdo que performam consistentemente, independente de plataforma ou nicho. Dominar os 5 garante variedade sem perder eficácia.

### Como funciona
```
TIPO 1 — O Ensaio de Opinião:
├── Formato: "Por que [crença contrária ao senso comum]"
├── Estrutura: Tese → 3 argumentos → Contra-argumento → Síntese
├── Exemplo: "Por que storytelling não é a habilidade mais importante de um escritor"
└── Quando usar: Para estabelecer perspectiva e autoridade

TIPO 2 — A Lista Tática:
├── Formato: "X coisas que [persona] deveria fazer para [resultado]"
├── Estrutura: Intro do problema → Lista numerada com 1 frase por item → CTA
├── Exemplo: "7 erros que destroem headlines antes do segundo parágrafo"
└── Quando usar: Para SEO, compartilhamento e valor imediato

TIPO 3 — A Lição de Erro:
├── Formato: "O erro que cometi tentando [objetivo] e o que aprendi"
├── Estrutura: O erro → A consequência real → O que funciona em vez disso
├── Exemplo: "Escrevi 500 posts antes de descobrir que estava escolhendo os temas errados"
└── Quando usar: Para criar identificação e construir confiança

TIPO 4 — O Antes/Depois:
├── Formato: Mostra a transformação com contraste claro
├── Estrutura: Antes (problema visível) → Depois (solução visível) → Como chegar lá
├── Exemplo: Reescreva o mesmo parágrafo antes e depois de aplicar o framework
└── Quando usar: Para demonstrar valor de forma instantânea

TIPO 5 — A Pergunta Provocadora:
├── Formato: Uma questão que força reconsideração de algo assumido
├── Estrutura: Pergunta → Contexto que a torna urgente → Implicação
├── Exemplo: "Se cada post que você escreveu sumisse hoje, o que você perderia?"
└── Quando usar: Para engajamento, comentários e reflexão
```

### Quando usar
- Planejamento de calendário editorial
- Diagnóstico de variedade no conteúdo publicado
- Escolha de formato antes de escrever

## Framework 3: O Diagnóstico do Writer's Problem

### O que é
O Writer's Problem é o erro fundamental de quem escreve: escrever do que você quer dizer, não do que o leitor precisa ouvir. Cole tem um sistema de diagnóstico.

### Como funciona
```
TESTE DO WRITER'S PROBLEM:
└── Leia sua abertura. Qual das seguintes ela é?

ESCRITOR-CÊNTRICA (sintoma do problema):
├── Começa com "Eu" ou "Nós"
├── Fala sobre sua perspectiva antes do problema do leitor
├── Pressupõe que o leitor já sabe por que deveria se importar
└── Exemplo: "Hoje quero compartilhar o que aprendi sobre headlines..."

LEITOR-CÊNTRICA (solução):
├── Começa com o problema que o leitor reconhece em si mesmo
├── Usa "você" antes de "eu" (ou nunca usa "eu")
├── O leitor sente "isso é sobre mim" nos primeiros 5 segundos
└── Exemplo: "A maioria dos posts que você escreve não performa porque..."

CORREÇÃO:
├── Identifique o problema real que o leitor tem
├── Escreva a abertura com esse problema em perspectiva do leitor
└── Só depois traga sua perspectiva como evidência
```

### Quando usar
- Revisão de qualquer peça escrita antes de publicar
- Reescrita de hooks e aberturas
- Avaliação de emails e newsletters

## Framework 4: O Sistema de Distribuição de Conteúdo

### O que é
Cole argumenta que criar conteúdo sem distribuição é tosse numa floresta. O sistema de distribuição garante que o conteúdo atinja quem pode se beneficiar dele.

### Como funciona
```
CAMADA 1 — Criação Original:
└── 1 ensaio longo por semana (newsletter/blog) = Conteúdo mãe

CAMADA 2 — Fragmentação:
├── Do ensaio longo → 5 posts curtos (1 insight por post)
├── Do ensaio longo → 1 thread (sequência linear dos pontos principais)
└── Dos melhores posts → Variações testadas

CAMADA 3 — Reutilização:
├── Posts que performaram bem → Repostar 90 dias depois (nova audiência não viu)
├── Posts antigos + novo contexto → "Escrevi isso X meses atrás. Ainda acredito porque..."
└── Série de posts relacionados → Compila em um ensaio longo

CICLO:
└── Ensaio → Posts → Thread → Variações → Reposts → Novo Ensaio
```

### Quando usar
- Planejamento de distribuição de conteúdo
- Maximização de ROI de cada peça criada
- Construção de consistência editorial com menor esforço criativo

## Framework 5: O Método de Criação de Categoria

### O que é
Inspirado por Category Pirates (que co-fundou), Cole ensina como criar uma nova categoria de produto/ideia em vez de competir em categoria existente.

### Como funciona
```
PASSO 1 — Diagnóstico do Problema de Categoria:
├── Em qual categoria existente você está tentando competir?
├── Quem são os incumbents dessa categoria?
└── Você pode ser top 3 nessa categoria? Se não, continue.

PASSO 2 — Identificação do Problema de Diferente Ordem:
├── O que o incumbente maximiza que você não maximiza?
├── O que você maximiza que o incumbente ignora?
└── Existe um grupo de pessoas que prefere o que você maximiza?

PASSO 3 — Nomeação da Nova Categoria:
├── A categoria deve ser: [Adjetivo diferenciador] + [Categoria existente]
├── Exemplo: "Ensaio Atômico" (vs. "post de blog")
├── O nome da categoria educa sobre a diferença
└── Testar: Uma pessoa que não te conhece entende a diferença?

PASSO 4 — Educação do Mercado:
├── Crie conteúdo que explica POR QUE a nova categoria existe
├── Mostre o problema que a categoria antiga tem
├── Demonstre como a nova categoria resolve diferente
└── Convide outros a usar a linguagem da nova categoria
```

### Quando usar
- Posicionamento de produto ou serviço
- Diferenciação em mercados saturados
- Criação de framework de conteúdo proprietário

## Checklist de Aplicação de Frameworks Cole

### Para cada peça de conteúdo (1-1-1 Check):
- [ ] Qual é o 1 problema que este texto resolve? (Definido antes de escrever)
- [ ] Qual é a 1 solução? (Específica, não vaga)
- [ ] Qual é a 1 ação? (Imediata e concreta)
- [ ] A abertura começa com o problema do leitor? (Não com "eu")
- [ ] Cada frase tem razão de existir? (Remove as que não têm)
- [ ] Tem 500 palavras ou menos? (Para formatos atômicos)

### Para estratégia de conteúdo:
- [ ] Tenho os Big 5 tipos na minha rotação?
- [ ] Estou fragmentando meu conteúdo mãe para distribuição?
- [ ] Meus melhores posts estão sendo reaproveitados?
- [ ] Estou construindo categoria ou competindo em categoria?
