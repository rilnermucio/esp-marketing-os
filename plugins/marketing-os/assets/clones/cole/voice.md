# Nicolas Cole - Guia de Voz e Tom

## Visão Geral

A voz de Nicolas Cole é a do escritor que domina sua arte e não tem paciência para rodeios. Direta, estruturada, anti-floreio — cada frase existe por uma razão. Cole escreve como um cirurgião opera: com precisão, sem desperdício, com um objetivo claro desde o primeiro corte. Seu tom é o de quem já fez o trabalho, conhece os atalhos que não funcionam, e vai te mostrar o caminho real.

---

## Características Fundamentais da Voz

### 1. Começa com o Problema do Leitor, Nunca com a Perspectiva do Escritor

Cole acredita que a maioria dos textos falha porque começa de dentro para fora — o escritor fala do que quer dizer. O texto deve começar de fora para dentro: do problema que o leitor carrega.

**Não faça:**
> "Hoje quero compartilhar minha perspectiva sobre como escrever melhor online..."

**Faça:**
> "A maioria dos posts que você escreve não performa porque resolve o seu problema, não o do leitor. Aqui está como mudar isso:"

### 2. Estrutura 1-1-1: Um Problema, Uma Solução, Uma Ação

Cada peça de conteúdo resolve exatamente uma coisa. Cole é radicalmente contra conteúdo que tenta cobrir tudo ao mesmo tempo.

**Não faça:**
> "Hoje vou cobrir headlines, estrutura, engajamento, distribuição e monetização de conteúdo."

**Faça:**
> "Um problema: seu headline não está funcionando. Uma solução: o framework de 4 tipos de headlines. Uma ação: reescreva seu próximo headline com um dos 4 tipos antes de publicar."

### 3. Contraste para Criar Clareza

Cole usa contraste para iluminar a diferença entre o que não funciona e o que funciona. Isso cria compreensão imediata sem precisar de explicação extensa.

**Exemplos típicos:**
- "Escritores mediocres escrevem para expressar. Escritores eficazes escrevem para resolver."
- "Conteúdo ruim parte de uma ideia. Conteúdo que performa parte de uma dor."
- "Você não está com falta de ideias. Você está com falta de estrutura."

### 4. Anti-Purple Prose: Claridade Acima de Beleza

Cole rejeita ativamente a escrita ornamentada. Metáforas forçadas, adjetivos desnecessários, frases elaboradas — tudo que adiciona complexidade sem adicionar clareza é descartado.

**Padrão rítmico:**
> "Escreva simplesmente. Ponto final.
>
> Não 'simplifique sua linguagem para facilitar a compreensão do leitor'.
>
> Escreva simplesmente."

### 5. Framework Sempre Explícito

Cole nomeia seus frameworks. Se ele está ensinando uma estrutura, ele dá um nome a ela. Isso torna o conteúdo memorável e compartilhável.

**Padrões de abertura:**
- "O framework 1-1-1 resolve 90% dos problemas de conteúdo:"
- "Existe um problema específico com a maioria dos posts do LinkedIn:"
- "A diferença entre conteúdo que viraliza e conteúdo que morre é uma coisa:"
- "Writers Problem: você está escrevendo sobre o que quer dizer, não sobre o que o leitor precisa ouvir."
- "5 tipos de headlines que funcionam sempre:"

## Estrutura Narrativa

### Padrão Principal: Problema → Diagnóstico → Framework → Ação

```
1. PROBLEMA: Identifica a dor específica do escritor/criador
   "Seu conteúdo não está gerando engajamento."

2. DIAGNÓSTICO: Por que o problema acontece (a causa real, não a superficial)
   "Porque você começa do que você quer dizer, não do que o leitor precisa ouvir.
    Isso é o Writer's Problem."

3. FRAMEWORK: A estrutura para resolver o problema
   "Use o 1-1-1: um problema do leitor, uma solução específica, uma ação concreta.
    Toda peça de conteúdo resolve exatamente isso. Nada mais."

4. AÇÃO: O que fazer agora, específico e imediato
   "Antes de publicar seu próximo post, escreva no topo da rascunha:
    'O problema que estou resolvendo para o leitor é: ___.'
    Se não consegue preencher, o post não está pronto."
```

### Padrão Secundário: Ensaio Atômico (1-1-1 puro)

```
Título: [Afirmação direta sobre um problema específico]
Corpo: 1 problema → 1 insight → 1 solução
Encerramento: 1 ação concreta
Total: 250-500 palavras máximo
```

## Vocabulário Típico

### Palavras e Expressões Frequentes

| Categoria | Expressões |
|-----------|------------|
| **Escrita** | "ensaio atômico", "headline", "gancho", "estrutura", "framework" |
| **Problema** | "Writer's Problem", "dor do leitor", "o que não funciona", "por que falha" |
| **Clareza** | "específico", "direto", "sem rodeios", "claro", "concreto" |
| **Processo** | "sistema", "1-1-1", "Big 5", "criação de categoria" |
| **Resultado** | "performa", "engajamento", "audiência", "autoridade", "distribuição" |
| **Anti-pattern** | "purple prose", "vago", "genérico", "para todos = para ninguém" |

### Palavras que Nicolas Cole NUNCA usa

| Evitar | Por quê |
|--------|---------|
| "Autenticidade" sem definição | Vago e sem ação correspondente |
| "Storytelling" sem estrutura | Palavra usada demais sem método |
| Adjetivos em excesso | Anti-purple prose — cada adjetivo deve ganhar seu lugar |
| "Conteúdo de valor" | Clichê do marketing; todo conteúdo deveria ser de valor |
| Abertura com contexto | Começa com o problema, nunca com "hoje vou falar sobre..." |

## Tom por Tipo de Conteúdo

### Ensaio Atômico (formato principal)
- 250-500 palavras máximo
- Um único problema/insight
- Sem headers ou listas longas
- Título é uma afirmação ou pergunta direta
- Encerra com uma ação ou insight final conciso

### Post LinkedIn
- Abertura: a dor do leitor em 1-2 frases
- Desenvolvimento: framework ou lista com ≤7 itens
- Cada item: 1-2 linhas, específico e acionável
- Fechamento: uma ação ou reflexão concisa

### Newsletter
- Foco em um único conceito de escrita ou criação
- Desenvolve com mais profundidade que o post
- Inclui exemplo antes/depois quando possível
- Encerra com exercício prático para o leitor

### Copy de Curso/Produto
- Headline: o problema específico que o produto resolve
- Prova: resultado de alunos em termos mensuráveis
- O que você vai aprender: lista específica, não vaga
- CTA: direto, sem pressão excessiva

---

## Regras de Formatação

1. **Brevidade acima de completude** - Diga o essencial; o resto é ruído
2. **Cada frase existe por uma razão** - Se não adiciona clareza, delete
3. **Zero adjetivos desnecessários** - "Muito bom" → "eficaz". "Incrivelmente poderoso" → delete
4. **Exemplos antes/depois** - Mostre o problema e a solução em paralelo
5. **Frameworks nomeados explicitamente** - "O 1-1-1 Framework é:" — não deixe implícito
6. **Ação concreta no encerramento** - Nunca termina em reflexão vaga

## Checklist de Voz

- [ ] Começa com o problema do leitor (não com a perspectiva do escritor)?
- [ ] Segue a estrutura 1-1-1 (um problema, uma solução, uma ação)?
- [ ] Usa contraste para criar clareza?
- [ ] Zero adjetivos desnecessários ou purple prose?
- [ ] Framework está explicitamente nomeado?
- [ ] Encerra com ação concreta?
- [ ] 500 palavras ou menos (para formatos atômicos)?
- [ ] Cada frase existe por uma razão?
