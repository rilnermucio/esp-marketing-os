# PARTE I: A CIÊNCIA DO DESIGN INSTRUCIONAL

## 1.1 Neurociência da Aprendizagem

### Como o Cérebro Aprende

```
ESTÍMULO → PROCESSAMENTO → CONSOLIDAÇÃO → APLICAÇÃO

    Conteúdo          Memória de         Sono/Revisão        Prática
    Visual/           Trabalho           Espaçada            Deliberada
    Auditivo          (limitada!)        (consolidação)      (maestria)
```

#### O Problema da Carga Cognitiva (Sweller, 2024-2026)

```
MEMÓRIA DE TRABALHO: LIMITADA A 3-7 ELEMENTOS SIMULTÂNEOS

┌─────────────────────────────────────────────────────────────┐
│                   CARGA COGNITIVA TOTAL                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  INTRÍNSECA      EXTRÍNSECA      GERMÂNICA                  │
│  (complexidade   (design ruim,   (processamento             │
│   do conteúdo)   distrações)     relevante)                 │
│                                                              │
│  [████████]      [████]          [████████████]              │
│  Não reduzível   REDUZIR!        MAXIMIZAR!                 │
│                                                              │
│  LIMITE DA MEMÓRIA DE TRABALHO ──────────────────── 100%    │
└─────────────────────────────────────────────────────────────┘

IMPLICAÇÕES PARA INFOPRODUTOS:
1. Chunking: dividir em pedaços de 3-7 informações
2. Eliminar distrações visuais e textuais
3. Sequenciamento lógico (simples → complexo)
4. Scaffolding: construir sobre conhecimento prévio
5. Redundância nociva: não repetir mesma info em texto + áudio + slide
```

### Sete Princípios Científicos de Aprendizagem Efetiva

```
1. ACTIVE RECALL (Recuperação Ativa)
   ├── Forçar o aluno a lembrar, não apenas revisar
   ├── Quizzes > Releitura
   ├── Efeito testing: recall fortalece memória 80% mais que revisão
   └── APLICAÇÃO: Quiz ao final de cada módulo

2. SPACED REPETITION (Repetição Espaçada)
   ├── Revisão em intervalos crescentes (1d, 3d, 7d, 14d, 30d)
   ├── Combate a curva do esquecimento
   └── APLICAÇÃO: Emails de revisão, resumos semanais

3. INTERLEAVING (Intercalação)
   ├── Misturar tópicos ao invés de blocar
   ├── Dificulta no momento, fortalece longo prazo
   └── APLICAÇÃO: Exercícios misturando conceitos de módulos diferentes

4. ELABORATIVE INTERROGATION (Interrogação Elaborativa)
   ├── Perguntar "por quê?" e "como?"
   ├── Conectar novo com existente
   └── APLICAÇÃO: Exercícios de reflexão em cada aula

5. DUAL CODING (Codificação Dupla)
   ├── Combinar visual + verbal
   ├── Dois canais de memória = retenção dobrada
   └── APLICAÇÃO: Diagramas + explicação, infográficos + texto

6. CHUNKING (Fragmentação)
   ├── 3-7 informações por unidade
   ├── Respeitar limites da memória de trabalho
   └── APLICAÇÃO: Módulos de 5-15 minutos

7. RETRIEVAL PRACTICE (Prática de Recuperação)
   ├── Testar ANTES de ensinar o próximo módulo
   ├── Errar faz parte (desirable difficulties)
   └── APLICAÇÃO: Pré-teste + pós-teste por módulo
```

### Implicações Para Infoprodutos

| Princípio | O que fazer | O que NÃO fazer |
|-----------|------------|-----------------|
| Necessidade de saber | Mostrar resultado antes de ensinar | Começar com teoria pura |
| Autoconceito | Dar opções e caminhos | Ser excessivamente diretivo |
| Experiência | Pedir exemplos pessoais | Ignorar conhecimento prévio |
| Prontidão | Resolver a dor urgente primeiro | Seguir ordem "acadêmica" |
| Orientação | Exercícios práticos imediatos | Aulas puramente conceituais |
| Motivação | Conectar com objetivos pessoais | Usar apenas pressão externa |

## 1.3 Aprendizagem Transformacional (Mezirow)

### Do Informacional ao Transformacional

```
┌─────────────────────────────────────────────────────────────┐
│              INFORMACIONAL vs TRANSFORMACIONAL                │
├────────────────────────┬────────────────────────────────────┤
│     INFORMACIONAL      │        TRANSFORMACIONAL            │
├────────────────────────┼────────────────────────────────────┤
│ Objetivo: transmitir   │ Objetivo: mudar comportamento/    │
│ conhecimento           │ identidade                         │
│ Preço: R$ 47-497       │ Preço: R$ 497-25.000+             │
│ Formato: self-paced    │ Formato: ao vivo + comunidade     │
│ Conclusão: 10-20%      │ Conclusão: 85-90%                 │
│ Resultado: "aprendi"   │ Resultado: "me transformei"        │
│ Conteúdo = 80% valor   │ Conteúdo = 30-40% valor           │
│ Diferencial: qualidade │ Diferencial: acesso + accountability│
│ Escalável: SIM         │ Escalável: LIMITADO                │
│ Defensável: BAIXO      │ Defensável: ALTO                  │
│ (copiável)             │ (baseado em experiência)           │
└────────────────────────┴────────────────────────────────────┘
```

### As 4 Fases da Transformação (Mezirow Adaptado)

```
FASE 1: DILEMA DESORIENTADOR
├── Desafiar a visão de mundo do aluno
├── Mostrar que o jeito atual não funciona
├── Criar tensão construtiva
└── EXEMPLO: "Você está perdendo R$ 10k/mês por não saber isso"

FASE 2: REFLEXÃO CRÍTICA
├── Aluno examina suas crenças e valores
├── Questionar "por que eu faço assim?"
├── Autoavaliação honesta
└── EXEMPLO: Exercício de diagnóstico da situação atual

FASE 3: DESENVOLVIMENTO DE NOVA PERSPECTIVA
├── Engajamento com comunidade
├── Novos frameworks e modelos mentais
├── Exposição a exemplos e cases
└── EXEMPLO: Aprender com pares que já alcançaram o resultado

FASE 4: INTEGRAÇÃO
├── Novas perspectivas integradas na prática
├── Mudança de comportamento real
├── Nova identidade consolidada
└── EXEMPLO: Implementar e obter resultados mensuráveis
```

### Seis Métodos de Ensino Transformacional

| Método | Descrição | Aplicação em Infoproduto |
|--------|-----------|--------------------------|
| **Visão compartilhada** | Estabelecer visão coletiva para o curso | Aula de boas-vindas com missão do programa |
| **Modelagem e maestria** | Demonstrar e praticar | Estudos de caso + exercícios hands-on |
| **Desafio intelectual** | Encorajar e provocar | Perguntas reflexivas, debates na comunidade |
| **Atenção personalizada** | Feedback individual | Correções, mentorias, check-ins |
| **Lições experienciais** | Transcender a sala de aula | Projetos reais, desafios práticos |
| **Reflexão** | Oportunidades de reflexão | Diários de aprendizado, autoavaliação |

## 1.4 Taxonomia de Bloom: Níveis de Aprendizado

### Hierarquia de Objetivos Cognitivos

```
                    ┌─────────┐
                    │  CRIAR  │  ← Gerar, planejar, produzir
                   ┌┴─────────┴┐
                   │  AVALIAR   │  ← Julgar, criticar, decidir
                  ┌┴───────────┴┐
                  │  ANALISAR    │  ← Diferenciar, organizar, atribuir
                 ┌┴─────────────┴┐
                 │   APLICAR      │  ← Executar, implementar, usar
                ┌┴───────────────┴┐
                │   COMPREENDER    │  ← Interpretar, classificar, resumir
               ┌┴─────────────────┴┐
               │     LEMBRAR        │  ← Reconhecer, listar, definir
               └───────────────────┘

NÍVEL INFERIOR: Informacional (R$ 47-197)
NÍVEL MÉDIO: Aplicacional (R$ 297-997)
NÍVEL SUPERIOR: Transformacional (R$ 997-25.000+)
```

### Aplicação em Design de Infoprodutos

| Nível Bloom | Tipo de Conteúdo | Tipo de Exercício | Exemplo |
|-------------|-----------------|-------------------|---------|
| **Lembrar** | Glossário, lista, resumo | Quiz múltipla escolha | "O que é CAC?" |
| **Compreender** | Explicação, comparação | Paráfrase, resumo | "Explique em suas palavras" |
| **Aplicar** | Tutorial, passo a passo | Exercício guiado | "Calcule o CAC do seu negócio" |
| **Analisar** | Estudo de caso | Análise comparativa | "Compare estes 3 funis" |
| **Avaliar** | Cenários, debates | Peer review, julgamento | "Avalie esta landing page" |
| **Criar** | Projeto final | Produção original | "Crie seu funil completo" |

## 5.3 Slides Efetivos (Design Instrucional)

### Regras de Design de Slides

```
REGRA 6x6:
├── Máximo 6 palavras por linha
├── Máximo 6 linhas por slide
├── Restante → narração por voz
└── Slide = apoio visual, NÃO teleprompter

HIERARQUIA VISUAL:
├── Título: 24-32pt, bold
├── Subtítulo: 18-24pt, regular
├── Corpo: 16-20pt, light
└── Notas: 12-14pt, muted color

REGRA DO CONTRASTE:
├── Fundo escuro → texto claro
├── Fundo claro → texto escuro
├── NUNCA: fundo complexo + texto sobreposto
└── Uma imagem de fundo OU texto, não ambos

TIPOS DE SLIDE:

1. SLIDE DE TÍTULO
   ├── Uma frase impactante
   └── Sem bullets, sem listas

2. SLIDE DE CONCEITO
   ├── Diagrama ou framework
   ├── Mínimo texto
   └── Explicação por voz

3. SLIDE DE DADOS
   ├── Um gráfico ou tabela
   ├── Destaque do insight principal
   └── Fonte citada

4. SLIDE DE COMPARAÇÃO
   ├── Antes vs Depois
   ├── Certo vs Errado
   └── Opção A vs Opção B

5. SLIDE DE AÇÃO
   ├── "Faça isso agora"
   ├── 1-3 passos claros
   └── Checklist visual
```

## 8.4 Precificação de Consultoria

### Modelos de Cobrança

```
POR PROJETO (mais seguro para ambos):
├── Escopo fechado, preço fixo
├── Bom quando: escopo é claro
├── Risco: scope creep (expandir escopo sem ajustar preço)
└── Proteção: contrato com escopo detalhado e adendos

POR HORA (mais flexível):
├── Valor/hora × horas estimadas
├── Bom quando: escopo é incerto
├── Risco: incentivo a trabalhar mais lento
└── Proteção: estimativa de horas com cap

RETAINER MENSAL (receita previsível):
├── Valor fixo mensal, renovação automática
├── Bom quando: relação contínua
├── Risco: acomodação de ambos os lados
└── Proteção: revisão trimestral de escopo

VALUE-BASED PRICING (mais lucrativo):
├── Preço baseado no VALOR gerado, não no tempo
├── Bom quando: ROI é mensurável
├── Risco: resultado depende do cliente também
└── Proteção: meta realista + cláusula de esforço mútuo

```
ETAPA 1: PESQUISA DE MERCADO

├── Gaps no mercado (o que falta?)

ETAPA 2: PESQUISA COM AUDIÊNCIA
├── Enquete em redes sociais (stories, LinkedIn)
├── Pesquisa por email (Google Forms)
├── Conversas 1:1 com potenciais clientes (10-20)

└── Resultado: "Meu público QUER isso? SIM/NÃO"

ETAPA 3: PRÉ-VENDA
├── Criar página de vendas (mesmo sem o produto)
├── Oferecer condição especial de pré-venda
├── Aceitar pagamento (reembolsável se não entregar)
├── Meta mínima: 10-30 pré-vendas
└── Resultado: "Pessoas PAGAM por isso? SIM/NÃO"

ETAPA 4: MVP (Minimum Viable Product)
├── Criar versão mínima do produto
├── Entregar para primeiros compradores (beta testers)
├── Coletar feedback detalhado
├── Iterar baseado em dados reais
└── Resultado: "O produto FUNCIONA? SIM/NÃO"

têm 3x mais chance de sucesso no lançamento.
```

## 13.7 Erros Fatais na Criação de Infoprodutos

### Os 15 Erros Mais Comuns (e Como Evitar)

```
ERROS DE ESTRATÉGIA:

1. CRIAR SEM VALIDAR
   ├── Erro: gastar meses criando algo que ninguém quer
   ├── Solução: pré-venda antes de criar (mínimo 10 vendas)
   └── Regra: "Não crie o que ninguém pediu"

2. PREÇO BASEADO EM CUSTO (não em valor)
   ├── Erro: "Gastei R$ 5k, vou vender por R$ 97"
   ├── Solução: preço = 10% do valor da transformação
   └── Regra: "Cobre pelo resultado, não pelo esforço"

3. TENTAR AGRADAR TODO MUNDO
   ├── Erro: curso genérico demais
   ├── Solução: nicho específico + persona clara
   └── Regra: "Quem fala para todos não fala para ninguém"

ERROS DE CONTEÚDO:

4. EXCESSO DE TEORIA
   ├── Erro: 80% teoria, 20% prática
   ├── Solução: inverter (20% teoria, 80% prática)
   └── Regra: "Adultos aprendem fazendo, não ouvindo"

5. AULAS LONGAS DEMAIS
   ├── Erro: aulas de 60+ minutos
   ├── Solução: microlearning (5-15 min por aula)
   └── Regra: "Chunking: 3-7 informações por unidade"

6. NÃO TER QUICK WIN
   ├── Erro: resultado só no final do curso
   ├── Solução: primeira vitória nas primeiras 2 horas
   └── Regra: "Time to first value < 24 horas"

7. COPIAR CONTEÚDO GENÉRICO
   ├── Erro: conteúdo que qualquer IA gera
   ├── Solução: experiência própria, cases, opinião, framework único
   └── Regra: "IA gera informação, humanos geram transformação"

ERROS DE PRODUÇÃO:

8. ÁUDIO RUIM
   ├── Erro: eco, ruído, volume baixo
   ├── Solução: microfone USB + ambiente acústico
   └── Regra: "Aluno perdoa vídeo ruim, não áudio ruim"

9. NÃO TER MATERIAIS DE APOIO
   ├── Erro: só vídeo, sem workbook/exercícios
   ├── Solução: workbook + checklists + templates
   └── Regra: "Materiais aumentam retenção em 40%"

10. PERFECCIONISMO
    ├── Erro: não lançar até estar "perfeito"
    ├── Solução: MVP primeiro, iterar depois
    └── Regra: "Done is better than perfect"

ERROS DE LANÇAMENTO:

11. NÃO TER AUDIÊNCIA
    ├── Erro: criar produto antes de ter público
    ├── Solução: construir lista/comunidade primeiro (mín 1.000)
    └── Regra: "Primeiro audiência, depois produto"

12. PÁGINA DE VENDAS FRACA
    ├── Erro: foco em features, não em benefícios
    ├── Solução: headline + dor + transformação + prova social
    └── Regra: "Venda o resultado, não o produto"

13. NÃO TER GARANTIA
    ├── Erro: sem garantia = barreira de compra
    ├── Solução: garantia de 7-30 dias (obrigatório por lei)
    └── Regra: "Garantia aumenta vendas e reduz reembolso"

ERROS PÓS-VENDA:

14. ONBOARDING RUIM
    ├── Erro: aluno compra e fica perdido
    ├── Solução: sequência de onboarding (3-7 emails)
    └── Regra: "Primeiras 48h definem se vai completar"

15. NÃO COLETAR DEPOIMENTOS
    ├── Erro: não pedir feedback/depoimento
    ├── Solução: pesquisa pós-módulo + pedido de depoimento
    └── Regra: "Depoimentos são o marketing mais poderoso"
```

  └── PRODUTO RECOMENDADO: Workshop ou Challenge + curso simples

  TEMPO LIMITADO → Produtos escaláveis:
  ├── Curso self-paced

  ├── Cohort-based course

  ├── Mini-curso self-paced

  ├── Curso completo

# PARTE XVIII: PRICING STRATEGY DEEP

Pricing de infoproduto é arte + ciência. Aqui frameworks específicos.

## 18.1 Anchoring (preço de comparação)

```
PRINCÍPIO:
Cliente avalia preço por contexto, não absoluto.

APLICAÇÃO EM SALES PAGE:

ANTES (sem anchor):
"Este curso custa R$1.997"
└── Cliente pensa: "É caro?"

DEPOIS (com anchor):
"Mentoria 1:1 comigo: R$5.000/mês.
Curso completo gravado: R$1.997 (uma vez).
Ebook básico: R$97."
└── Cliente vê: R$1.997 está "no meio", parece justo

VARIAÇÕES:

VS PREÇO HORA-HUMANA:
"Em 12 horas comigo no consultório, R$2.400 (R$200/h).
Em 12 horas neste curso, mesmo conteúdo, R$497."

VS PREÇO ALTERNATIVA:
"Pós-graduação em [tema]: 24 meses, R$36.000.
Este curso: 30 dias, R$497, mesmo conteúdo prático."

VS PREÇO PRÓPRIO PRODUTO ANTERIOR:
"V1 deste curso: R$2.997.
V2 (esta versão): R$1.997. Promoção lançamento."
```

## 18.2 Decoy Pricing (3 opções)

```
ESTRUTURA:

OPÇÃO A: Básico (R$497)
├── Curso completo gravado
├── Sem comunidade
└── Sem suporte

OPÇÃO B: Pro (R$997) ← QUEREMOS QUE COMPRE ESTA
├── Curso completo gravado
├── Comunidade Telegram
├── 4 calls live
└── Bonus: ebook X

OPÇÃO C: Premium (R$2.497)
├── Tudo do Pro
├── Mentoria 1:1 mensal (3 sessões)
└── Acesso a curso secundário

PSICOLOGIA:
├── A sozinha pareceria "limitada"
├── B sozinha pareceria "cara"
├── A + B + C: B parece "perfeita relação custo-benefício"
└── Resultado: ~70% escolhe B (anchoring + decoy)
```

## 18.3 Bundling vs Unbundling

```
BUNDLING (pacote):
├── 5 produtos juntos por preço único
├── Cliente sente que "leva mais"
├── Reduce decisão (1 escolha)
└── Use pra: aumentar ticket médio

UNBUNDLING (separado):
├── Vendas individuais
├── Cliente paga só pelo que quer
├── Reduz fricção pra entrada
└── Use pra: testes, validação

EXEMPLO:
BUNDLE: Curso + Ebook + Templates + Comunidade — R$997
UNBUNDLED:
├── Curso: R$497
├── Ebook: R$97
├── Templates: R$97
└── Comunidade: R$97/mês

Total se comprar tudo separado: ~R$700-900/ano
Bundle desconto sobre o total: signal de valor
```

## 18.4 Dynamic Pricing (lançamento vs evergreen)

```
LAUNCH PRICING:
├── Carrinho aberto 7-10 dias
├── Preço promocional
├── Após carrinho: volta ao "preço cheio" (3-5x maior)
└── Re-abertura: 6-12 meses depois

EVERGREEN PRICING:
├── Preço fixo permanente
├── Sem urgência sazonal
├── Pode ter "preço early bird" pra primeiros N alunos
└── Use pra: cursos perpétuos (não-PLF)

STRATEGY HÍBRIDA:
├── Lançamento sazonal (3-4x por ano)
├── Preço sempre "promocional" durante lançamento
├── Entre lançamentos: lista de espera
└── Mais comum em info-produtos BR
```

## 18.5 Pix Discount BR (estratégia comum)

```
LÓGICA:

CARTÃO 12X:
├── Plataforma cobra 8-13% (Hotmart, Kiwify)
├── Custo cliente: R$X
├── Você recebe: R$X * (1 - taxa)

PIX:
├── Plataforma cobra 1-2%
├── Custo cliente: R$X
├── Você recebe: R$X * (1 - taxa)
├── Diferença: 6-11% mais pra você

OFERTA TÍPICA:
"R$1.997 em 12x R$199 OU
R$1.797 à vista no Pix (10% desconto)"

WIN-WIN:
├── Cliente: paga menos
├── Você: recebe mais (mesmo com desconto)
├── Cobrança imediata (sem chargeback)
└── Aluno mais comprometido (já pagou tudo)

ADOÇÃO:
30-50% dos alunos optam por Pix se desconto >7%.
```

## 18.6 Pricing por Etapa do Funil

```
LEAD MAGNET (gratuito):
├── Mini-curso, ebook, checklist
├── Custo: R$0
└── Objetivo: gerar lead

TRIPWIRE (R$27-97):
├── Compra de baixo risco
├── Cliente compra primeiro (psicologia)
└── Objetivo: converter lead em cliente

CORE OFFER (R$497-1997):
├── Produto principal
├── Maior receita
└── Objetivo: monetização principal

UPSELL/CROSS-SELL (R$97-997):
├── No checkout ou pós-compra

# PARTE XX: CONAR + BR COMPLIANCE PARA INFOPRODUTOS

Infoprodutos BR têm 3 níveis de compliance: CONAR (publicidade), CDC (consumidor), e setoriais (MEC se aplicável).

## 20.1 CONAR Aplicado a Infoprodutos

```
PRINCIPAIS RISCOS CONAR EM INFOPRODUTOS:

CLAIM DE TRANSFORMAÇÃO:
├── "Você vai ganhar R$10k/mês" sem disclaimer = violation
├── Antes/depois sem "resultados podem variar" = violation
├── Solução: disclaimer obrigatório em sales page + ads

GUARANTEE INFLATION:
├── "Você TEM que ganhar X em Y dias" = problema
├── "Aprenderá X em Y dias" = OK
└── Diferença: aprendizado vs resultado financeiro

DEPOIMENTOS:
├── Cliente compensado precisa declarar
├── Foto do depoente precisa autorização
├── Texto não pode ser inventado/editado significativamente
└── Disclaimer: "depoimento real. Resultado individual pode variar."

COMPARAÇÕES:
├── "Melhor curso do Brasil" sem prova = violation
├── "Mais completo que [concorrente]" sem documentação = violation
└── Ranking citado precisa fonte verificável

CRIANÇAS:
├── Cursos pra menores: regras extras
├── Não pode estimular pedido a pais
└── Cuidado em conteúdo educacional infantil
```

## 20.2 CDC Específico pra Infoprodutos

```
ART. 49 - DIREITO DE ARREPENDIMENTO:
├── 7 dias mínimo (lei)
├── Conta da liberação de acesso (digital)
├── Reembolso INTEGRAL (incluindo taxas)
└── Sua garantia de 30 dias = padrão MOS, vai além

ART. 30-35 OFERTA E PUBLICIDADE:
├── Tudo na sales page vincula
├── Bonus prometido tem que entregar
├── Não pode ser "diferente" depois da compra
└── Mudanças de produto pós-compra precisam consent

ART. 39 PRÁTICAS ABUSIVAS:
├── "Compre agora ou nunca mais" forçando compra = problema
├── Aproveitar fragilidade emocional = problema
└── Falsa escassez ("3 vagas" repetida) = problema

ACESSO AO CURSO:
├── Direito do aluno: acesso indefinido (a menos que acordado)
├── Cancelamento de plataforma: aluno tem direito a continuidade
├── Mude de plataforma: comunique + ofereça alternativa
└── Documente acordos (sem letras miúdas)
```

## 20.3 MEC e Certificação Reconhecida

```
TIPOS DE CERTIFICAÇÃO BR:

LIVRE (sem MEC):
├── Curso "livre" em entidade não-credenciada
├── Certificado válido como "complementar"
├── NÃO equivale a curso técnico/superior
├── Mais comum em infoprodutos BR
└── 95% dos cursos online BR são "livres"

CURSO TÉCNICO (com MEC):
├── Credenciamento via SGSC (Sistema Nacional de Capacitação)
├── Reconhecido pelo MEC
├── Requisitos rigorosos (programa, carga, instrutor)
├── Diferencial competitivo grande
└── Custo: investimento em credenciamento

PÓS-GRADUAÇÃO (lato sensu):
├── Com MEC + parceria com IES (Instituição de Ensino Superior)
├── Equivale a especialização
├── Valor mais alto (R$5k-30k)
└── Diferencial premium

ESCOLHAS:

Curso prático (skills): "livre" é suficiente
Curso de carreira: "técnico MEC" se ambicioso
Curso premium executivo: "pós-graduação MEC" via parceria
Cursos rápidos/microlearning: "livre" sempre

DISCLAIMER OBRIGATÓRIO:
"Este é um curso livre. Não é equivalente a curso
técnico ou de graduação. Certificado emitido pela
[entidade] não tem validade reconhecida pelo MEC."

OMITIR ESSA INFO = engano (CONAR + CDC).
```

## 20.4 Compliance Quality Gate

```
ANTES DE LANÇAR INFOPRODUTO BR:

□ Sales page tem disclaimer "resultados podem variar"?
□ Garantia >= 7 dias (CDC) ou estendida (30+ padrão MOS)?
□ Depoimentos são reais com autorização?
□ Comparações com concorrentes têm prova?
□ Categoria regulada (saúde/financeiro) tem disclaimers obrigatórios?
□ Tipo de certificação está claro (livre vs MEC)?
□ Bonus prometidos podem ser entregues?
□ Sem práticas abusivas (CDC Art. 39)?
□ Política de privacidade LGPD-compliant?
□ Termos de uso disponíveis pre-compra?

Falha em qualquer um → ajustar antes de lançar.
```
