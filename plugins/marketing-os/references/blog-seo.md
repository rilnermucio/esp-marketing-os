## Pesquisa de Keywords

### Tipos de Keywords

| Tipo | Volume | Competição | Intenção | Exemplo |
|------|--------|------------|----------|---------|
| Head | Alto | Alta | Informacional | "marketing digital" |
| Middle | Médio | Média | Mista | "curso marketing digital" |
| Long-tail | Baixo | Baixa | Transacional | "curso marketing digital para iniciantes online" |

### Intenção de Busca (Search Intent)

| Intenção | Palavras-chave típicas | Tipo de conteúdo |
|----------|------------------------|------------------|
| Informacional | "o que é", "como fazer", "por que" | Guias, tutoriais, explicações |
| Navegacional | [marca], [produto] | Páginas institucionais |
| Transacional | "comprar", "preço", "melhor" | Landing pages, comparativos |
| Comercial | "review", "vs", "alternativas" | Reviews, comparações |

### Estrutura de Keyword Research

1. **Keyword seed** → Termo principal do nicho
2. **Expandir** → Variações, sinônimos, perguntas
3. **Analisar** → Volume, dificuldade, CPC
4. **Agrupar** → Por intenção e topic cluster
5. **Priorizar** → Quick wins vs long-term

### Ferramentas (usar via browser ou API quando disponível)

- Google Search (autocomplete, "People also ask")
- Google Trends
- AnswerThePublic
- Ubersuggest
- SEMrush / Ahrefs (se disponível)

## Estrutura de Artigo SEO

### Template de Artigo Otimizado

```markdown
# [Keyword Principal] + [Benefício/Modificador]


[Introdução - 100-150 palavras]
- Hook que conecta com a dor/desejo
- Promessa do que será entregue
- Keyword nas primeiras 100 palavras

## [H2 com Keyword Secundária]

[Conteúdo]
- Parágrafos curtos (2-3 frases)
- Bullet points para listas
- Keywords naturalmente distribuídas

### [H3 - Subtópico]

[Detalhar o ponto]

## [Próximo H2]

...

## FAQ / Perguntas Frequentes


### [Pergunta exata que as pessoas buscam]?

[Resposta direta em 40-60 palavras, depois expandir]

## Conclusão

[Resumo + CTA claro]
- Recapitular pontos principais
- Call-to-action específico
- Keyword final natural
```

### Comprimento Ideal por Tipo

| Tipo de Conteúdo | Palavras | Uso |
|------------------|----------|-----|
| Notícia/Atualização | 300-600 | Novidades, announcements |
| Post padrão | 1.000-1.500 | Dicas, listas, how-to simples |
| Guia completo | 2.000-3.000 | Tutoriais detalhados |
| Pillar content | 3.000-5.000+ | Guias definitivos, topic clusters |

## Otimização On-Page

### Checklist de SEO On-Page

#### Meta Tags
- [ ] **Title tag**: 50-60 caracteres, keyword no início
- [ ] **Meta description**: 150-160 caracteres, CTA incluído
- [ ] **URL slug**: Curta, com keyword, sem stop words

#### Headings
- [ ] **H1**: Único, com keyword principal
- [ ] **H2s**: Keywords secundárias, estrutura lógica
- [ ] **H3s**: Subtópicos, long-tails

#### Conteúdo
- [ ] Keyword density: 1-2% (natural, não forçado)
- [ ] Keyword nas primeiras 100 palavras
- [ ] Sinônimos e variações semânticas (LSI)
- [ ] Parágrafos curtos (máx 3 frases)
- [ ] Listas e bullets para escaneabilidade

#### Imagens
- [ ] **Alt text**: Descritivo com keyword quando relevante
- [ ] **Nome do arquivo**: descritivo-com-hifens.jpg
- [ ] **Compressão**: <100KB quando possível
- [ ] **Dimensões**: Especificadas no HTML

#### Links
- [ ] **Internos**: 3-5 links para conteúdo relacionado
- [ ] **Externos**: 2-3 links para fontes autoritativas
- [ ] **Anchor text**: Descritivo, não "clique aqui"

### Fórmulas de Title Tag

```
[Keyword Principal]: [Benefício] ([Ano])
Como [Ação] [Objeto]: Guia Completo [Ano]
[Número] [Assunto] que [Resultado] em [Tempo]
[Keyword]: O que é, Como Funciona e [Benefício]
[Keyword] vs [Alternativa]: Qual é Melhor?
```

### Fórmulas de Meta Description

```
Aprenda [o que]. Neste guia você vai descobrir [benefício 1], [benefício 2] e [benefício 3]. [CTA]!

Descubra [número] [assunto] para [resultado]. [Prova social ou dado]. Leia agora!

[Pergunta]? Explicamos tudo sobre [tema] e mostramos [benefício]. Confira!
```

## E-E-A-T

### Experience, Expertise, Authoritativeness, Trustworthiness

#### Experience (Experiência)
- Demonstrar experiência prática com o tema
- Incluir casos pessoais, screenshots, resultados
- "Depois de X anos fazendo Y, aprendi que..."

#### Expertise (Expertise)
- Credenciais do autor visíveis
- Bio completa com qualificações
- Conteúdo técnico preciso e atualizado

#### Authoritativeness (Autoridade)
- Links de fontes confiáveis
- Citações de especialistas
- Menções e backlinks de sites relevantes

#### Trustworthiness (Confiabilidade)
- Informações verificáveis
- Fontes citadas
- Página sobre, contato, política de privacidade
- HTTPS, design profissional

### Implementação E-E-A-T

```markdown
## Sobre o Autor

[Nome] é [credencial] com [X anos] de experiência em [área].
Já ajudou [número] [clientes/alunos] a [resultado].
[Link para LinkedIn/Portfolio]

## Fontes e Referências

1. [Estudo/Artigo] - [Instituição] ([Ano])
2. [Dado estatístico] - [Fonte]
```

## Tipos de Conteúdo

### 1. How-To / Tutorial

```markdown
# Como [Fazer X]: [Resultado] em [Tempo/Passos]

## O que você vai precisar
- Item 1
- Item 2

## Passo 1: [Ação]
[Instrução detalhada]
[Screenshot/imagem se aplicável]

## Passo 2: [Ação]
...

## Dicas de Expert
- Dica 1
- Dica 2

## Erros Comuns a Evitar
- Erro 1: [Por que evitar]
```

### 2. Listicle

```markdown
# [Número] [Assunto] para [Resultado] em [Ano]

## 1. [Item]
[Por que é importante]
[Como usar/aplicar]
**Destaque**: [Benefício principal]

## 2. [Item]
...
```

### 3. Comparativo

```markdown
# [Opção A] vs [Opção B]: Qual Escolher em [Ano]?

## Resumo Rápido
| Critério | Opção A | Opção B |
|----------|---------|---------|
| Preço | $X | $Y |
| Melhor para | [Caso] | [Caso] |

## O que é [Opção A]?
## O que é [Opção B]?

## Comparação Detalhada

### [Critério 1]
**Opção A**: [Análise]
**Opção B**: [Análise]
**Vencedor**: [Qual e por quê]

## Veredicto Final
```

### 4. Pillar Page (Topic Cluster Hub)

```markdown
# O Guia Definitivo de [Tema Principal]

## Índice
1. [Subtópico 1] → [Link para artigo cluster]
2. [Subtópico 2] → [Link para artigo cluster]
...

## O que é [Tema]?
[Definição completa]

## Por que [Tema] é Importante?
[Contexto e relevância]

## [Subtópico 1]: Visão Geral
[Resumo + link para artigo completo]

## [Subtópico 2]: Visão Geral
...

## Próximos Passos
[CTA + recursos adicionais]
```

### 5. Case Study

```markdown
# Como [Cliente/Eu] [Resultado] em [Tempo]: Case Study

## Resumo do Case
- **Desafio**: [Problema]
- **Solução**: [O que foi feito]
- **Resultado**: [Métricas]

## Contexto
[Situação inicial]

## O Desafio
[Problema detalhado]

## A Estratégia
### Fase 1: [Nome]
### Fase 2: [Nome]

## Os Resultados
[Dados, gráficos, screenshots]

## Lições Aprendidas
1. [Lição]
2. [Lição]

## Aplique na Sua Empresa
[CTA]
```

## Featured Snippets

### Tipos e Como Otimizar

| Tipo | Formato | Como Conseguir |
|------|---------|----------------|
| Parágrafo | Definição | Resposta direta em 40-60 palavras após H2/H3 |
| Lista | Bullet points | Lista com H2 + bullets ou H3s numerados |
| Tabela | Comparação | Tabela HTML/Markdown com headers claros |
| Vídeo | YouTube | Vídeo embedado com transcrição |

### Template para Snippet de Parágrafo

```markdown
## O que é [Keyword]?

[Keyword] é [definição direta em uma frase]. [Expansão em 2-3 frases que complementam]. [Contexto de uso ou importância].
```

### Template para Snippet de Lista

```markdown
## Como [Fazer X] em [Número] Passos

Para [fazer X], siga estes passos:

1. **[Passo 1]**: [Breve descrição]
2. **[Passo 2]**: [Breve descrição]
3. **[Passo 3]**: [Breve descrição]
```
